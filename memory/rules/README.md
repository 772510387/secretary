# Memory Rules

保存系统长期遵守的交易原则和风控边界。

## 需要沉淀

- 核心原则。
- A 股交易规则。
- 模拟盘规则。
- 风控阈值解释。
- 实盘禁用和启用条件。
- 模型权限边界。

## 写入要求

- 规则修改必须进入 `memory/proposals`。
- 通过人工确认后才能更新最终规则。
- 更新后需要写审计日志。

## MemoryWritePolicy 边界

`MemoryWritePolicy` 会把以下规则类写入判定为 `proposal_required`：

- 主板限制。
- T+1。
- 100 股整数。
- 8% 止损。
- 单股 40%。
- 实盘开关。
- broker 边界。

如果写入请求包含删除审计、写入密钥、绕过风控、写账户/订单或把模型输出变成直接订单，会被判定为 `reject`。

软阈值小幅调整只有在提供取值范围、证据引用且不削弱硬规则时才可以自动允许；实际落盘仍必须由 storage 层写入并保留审计。
