# Docs

这里存放项目需求、架构、运维、AI 协作和决策记录。

## 目录

- `requirements`：OpenClaw 原型阶段的原始需求记录。
- `requirement/`：整理后的需求和实现方案。
- `architecture/`：系统架构、模块关系、数据流和边界。
- `ops/`：部署、运行、实盘准备和事故处理。
- `ai/`：vibecoding 辅助资料。

## 维护要求

- 需求变化先写入 `docs/requirement` 或 `docs/architecture`。
- 架构决策要补充到 `docs/architecture/decision-records`。
- 新增模块时，必须在对应实现目录创建 `README.md`。
