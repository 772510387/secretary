# Secretary 架构落地工作拆分

生成日期：2026-06-12

## 推荐建设顺序

1. 项目基础设施
   - TypeScript 配置。
   - 配置加载。
   - 日志。
   - JSON 原子写入。
   - schema 校验。

2. 账户和模拟交易
   - 初始资金。
   - 持仓。
   - 交易流水。
   - T+1 可卖数量。
   - 100 股整数。
   - 主板过滤。

3. 风控
   - 单股 40%。
   - 8% 硬止损。
   - 单日亏损限制。
   - 禁买开关。
   - 熔断状态。

4. 行情
   - 腾讯实时行情。
   - 历史 K 线。
   - 指数行情。
   - 数据缓存。
   - provider 降级。

5. 小脑
   - 北京时间交易日历。
   - 盘中哨兵。
   - 固定闹钟。
   - 告警冷却。
   - 事件回放。

6. 记忆
   - 文件夹结构。
   - 读写策略。
   - 写入提案。
   - 自动归档。
   - 每日自省。

7. 大脑
   - `BrainProvider` 抽象。
   - OpenAI/Gemini/DashScope Qwen/mock provider。
   - 结构化输出校验。
   - 工具调用边界。

8. 研究适配器
   - TradingAgents-CN 作为可选研究能力。
   - 输出必须落为 `ResearchReport`。
   - 不进入交易执行。

9. 通知和入口
   - CLI 查询。
   - Webhook。
   - 告警消息。
   - 未来 UI/API。

10. 实盘预备
    - `BrokerAdapter` 抽象。
    - `PaperBroker`。
    - `ManualConfirmBroker`。
    - 未来 QMT/PTrade。

## MVP 验收

- 能维护一个模拟账户。
- 能拉实时行情。
- 能模拟买卖并写入流水。
- 能执行硬风控。
- 能定时生成盘前、午间、收盘、夜间报告。
- 能在异动时告警。
- 能把模型建议保存为提案，而不是直接改核心数据。
